# FIT Video Syncer & Trimmer 🚴⚡🎥

このプロジェクトは、自転車やランニングなどの走行ログデータ（`.fit` ファイル）を、撮影した動画ファイル（`.mp4` / `.lrv` / `.mov`）の録画時間に合わせて**クライアントサイド（ブラウザ内）だけで一瞬で同期・トリミング（切り出し）するツール**です。

*   **Webアプリ版 (GitHub Pages)**: [https://yuujikamura.github.io/fit-trimmer/](https://yuujikamura.github.io/fit-trimmer/)
*   **100% ローカル処理**: ファイルをサーバーにアップロードしないため、個人情報の保護はもちろん、巨大な動画ファイルでも通信データ量を消費せず超高速で処理できます。

---

## 🛠️ トリミングの仕組み (How it Works)

本ツールは**時刻（UTC: 協定世界時）**をキーにして、動画と走行データを同期しています。

1.  **動画メタデータの解析**:
    動画ファイルの先頭バイナリ（`moov -> mvhd` アトム）を高速スキャンし、そこに記録されている「撮影開始日時（UTC）」と「録画時間」をミリ秒単位で抽出します。
2.  **FITデータのスライス**:
    FITファイル内の各データレコードに含まれるタイムスタンプ（FITエポック秒数）をスキャンし、動画の撮影時間範囲に合致するレコードだけを切り出します。
3.  **走行データの自動補正（辻褄合わせ）**:
    *   **距離のリスタート**: 切り出された開始地点での累積距離を `0` メートルとし、以降の各レコードの距離から初期オフセットを減算します。これを行わないと、トリミング後に距離が「50km」などの途中からスタートしてしまいます。
    *   **概要サマリーの書き換え**: `session`（セッション全体）や `lap`（ラップ）の開始/終了時刻、走行時間、合計獲得距離などのサマリー情報を、切り出し後の実データに基づいて再計算・更新します。
    *   **CRCとサイズの再計算**: データを削ったことで変わるレコード部バイトサイズと、ヘッダーおよびファイル全体のCRC-16チェックサムを再計算し、Garmin ConnectやStrava等で「ファイル破損エラー」が起きないようファイルの整合性を完全に修復します。

## 🎥 MP4 と LRV の関係性 (MP4 vs LRV)

GoProなどのアクションカメラで撮影すると、本編の高画質動画（`.mp4`）と同時に、低解像度のプレビュー用動画（`.lrv` / Low Resolution Video）が生成されます。

*   **どちらを使っても結果は同じ**:
    `mp4` と `lrv` は全く同時に録画がスタートし、同時に終了するため、ファイル内に書き込まれている**作成日時や録画秒数は1ミリ秒の狂いもなく完全に同一**です。
*   **`.lrv` の使用を強く推奨**:
    本編の `mp4` はファイルサイズが 12GB など非常に大きくなりますが、 `lrv` は 1GB 程度（約12分の1）と非常に軽量です。ブラウザで読み込ませる際、 `lrv` の方が圧倒的に処理が軽く、ブラウザやPCのメモリ負荷を抑えて高速に処理が完了するため、 `lrv` をドロップして処理するのが最適です。

## ⏰ カメラの内蔵時計のズレと同期の問題

時刻ベースの同期における唯一かつ最大の弱点が、**「カメラの内蔵時計がズレていると、同期のタイミングもズレてしまう」**という問題です。

*   **GPSデバイス（FIT側）の正確さ**:
    Garmin等のサイコンやスマートウォッチは、GPS衛星から送られてくる超精密な原子時計の電波を受信して時刻をミリ秒単位で自動補正するため、記録される時刻は常に100%正確です。
*   **カメラ側の時計のズレやすさ**:
    GPSを内蔵していないカメラやGPS受信がオフのカメラは、手動設定された内蔵クオーツ時計に依存して録画日時を記録します。これは充電切れや時間経過で数分〜数十分ズレていくため、同期ズレの原因になります。
*   **予防策（裏技）**:
    撮影を開始する前に、カメラをメーカーのスマートフォンアプリ（GoPro Quik、Insta360アプリ等）に一度接続してください。接続された瞬間に、スマホの超正確な時刻（携帯電話基地局と同期された時刻）がカメラの内蔵時計に自動同期され、ズレを防ぐことができます。

## 💡 【応用編】緯度経度（GPS軌跡）を使った自動時間ズレ補正の可能性

「カメラの時計がズレていても、位置情報を使って自動補正できないか？」というのは、非常に高度で実用的なアイデアです。

### 1. 軌跡パターンマッチングによる自動同期の原理
もし動画ファイル側にも毎秒のGPS軌跡（緯度経度の時系列）が記録されている場合、以下のアルゴリズムで時間ズレを完全自動で計算・補正できます。
1.  **軌跡のスライド重ね合わせ**:
    動画側のGPSの移動軌跡（数十分分）を、FITファイル側の長いGPS軌跡の上に重ね合わせ、時間軸を1秒ずつスライドさせます。
2.  **最小誤差の探索**:
    動画のルートとFITのルートの「緯度経度の距離の差の合計（ズレ）」が最も小さくなる時間差（タイムオフセット）を探索します（空間的相互相関の計算）。
3.  **自動同期**:
    最も軌跡が一致したポイントの時間差を「カメラの時計のズレ」として自動認識し、そのズレを引いた正しい時刻でトリミングを実行します。これなら、カメラの時計が何時間ズレていても完全に自動同期します。

### 2. より実用的な「録画開始時の数秒間（点）」での補正
動画全体に毎秒のGPS軌跡を記録し続けるとカメラのバッテリー負荷が高くなります。しかし、**「録画開始時の数十秒間だけのGPS位置情報（および移動速度・方向）」**が動画に載っていれば、以下の方法でスマートに補正が可能です。
*   **測位誤差とルート重複の回避**:
    GPSの測位誤差（5〜20m）や、往復・周回ルートで同じ場所を2回通るケースを考慮し、単純な「開始時の1点」だけでなく、**「開始から数十秒間の移動の方向（方位ベクトル）と速度」**をFITデータと照合します。
    「その位置にいて、かつ時速〇kmで西に向かって動き出した瞬間」をFIT側のタイムラインから検索することで、動画の正確なGPS開始時刻を1点から特定し、同期を完了させることができます。

### 🔋 なぜカメラにGPSが乗っていないことが多いのか？（デバイスの制約）
「スマホのアプリから録画を開始すると、スマホのGPSデータをBluetooth経由でリアルタイムにカメラに送信し、動画にGPSを埋め込む」という機能（Insta360 GOシリーズ等でサポート）があります。

これは、超小型アクションカメラ（Insta360 GOシリーズなど）において、**「カメラ単体にGPSを内蔵する物理的コスト（制約）」**が非常に大きいために生まれた工夫です。
1.  **バッテリー消費**: GPSチップは衛星からの微弱な電波を常に受信・計算し続けるため消費電力が非常に大きく、極小バッテリーしか積めない超小型カメラでは撮影時間が極端に短くなってしまいます。
2.  **サイズと重量**: GPSアンテナとノイズシールドを積むと、コンセプトである「親指サイズ・軽量」が維持できなくなります。

そのため、スマホや専用リモコンなどの外部デバイスにGPS取得を「肩代わり」させ、接続経由で動画に位置情報を記録する設計になっています。
